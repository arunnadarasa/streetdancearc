
# Dance Move Tokens — Arc Testnet single-page demo

A street-dance merchandise/movement-licensing demo where users sign in with Google (Privy), pick a stablecoin (USDC / EURC / cirBTC), and call `log(token, amount, cid)` on a Solidity contract deployed to Arc Testnet via Circle Smart Contract Platform. USDC is the native gas token (6 decimals). One page, no router, no Lovable Cloud, no DB.

Note on "Spotify template": there is no prior Spotify-themed layout in this project. I'll apply a Spotify-inspired dark visual style (near-black background, vibrant green accent, rounded cards, bold display type) to the single page — purely presentational. If you meant a different existing template, tell me before I build.

## Scope

- ONE page at `/` (rewrite `src/routes/index.tsx`). No router additions, no auth pages.
- ONE Solidity contract (`contracts/DanceMoveTokens.sol`), Circle SCP deploy + Blockscout verify scripts.
- Privy embedded wallet as the ONLY auth/wallet layer, `defaultChain` + `supportedChains` = Arc Testnet.
- Multi-stablecoin: USDC / EURC / cirBTC chips; user approves ERC-20, then contract call.
- No Lovable Cloud, no Supabase, no backend routes. TanStack Start stays but the app is effectively SPA-shaped.

## Files to create / change

- `src/components/PrivyRoot.tsx` — SSR-safe Privy provider (lazy client-only import, gated by `<ClientOnly>`), configured with Google login, embedded wallets, `defaultChain`/`supportedChains`.
- `src/components/TreasuryCard.tsx` — reads `VITE_CIRCLE_TREASURY_ADDRESS`, copy button, faucet link; fallback message when empty.
- `src/components/TokenSwitcher.tsx` — three chips.
- `src/components/MintForm.tsx` — CID + amount inputs, approve + `log()` flow using Privy signer + viem.
- `src/components/TxToast.tsx` — arcscan tx link result.
- `src/routes/index.tsx` — rewrite placeholder; compose the above; Spotify-style dark UI; footer credit line.
- `src/routes/__root.tsx` — update `head()` title/description/og to "Dance Move Tokens".
- `src/data/contract.json` — placeholder stub committed so the UI compiles before deploy fills it in.
- `src/styles.css` — extend theme tokens (dark bg, green accent) without breaking existing shadcn tokens.

Contract + scripts
- `contracts/DanceMoveTokens.sol` — verbatim from spec (NatSpec credit included).
- `scripts/bootstrap-circle.mjs` — verbatim.
- `scripts/deploy-arc.mjs` — verbatim, plus preflight that exits with `Run: node scripts/bootstrap-circle.mjs` when `CIRCLE_TREASURY_WALLET_ID` or `CIRCLE_ENTITY_SECRET` is missing.
- `scripts/verify-arc.mjs` — verbatim.
- `bun add solc @privy-io/react-auth viem`  (and `@circle-fin/kit` only if you confirm you want the Bridge/Swap widget).

Secrets
- Ask you to paste: `CIRCLE_API_KEY`, `PRIVY_APP_ID`.
- Also exposed as `VITE_PRIVY_APP_ID` and (optional) `VITE_CIRCLE_KIT_KEY`, `VITE_CIRBTC_ADDRESS`, `VITE_ARC_RPC_URL`, `VITE_CIRCLE_TREASURY_ADDRESS` — Vite only ships `VITE_`-prefixed vars to the client.
- Auto-generated by bootstrap: `CIRCLE_ENTITY_SECRET`, `CIRCLE_TREASURY_WALLET_ID`, `VITE_CIRCLE_TREASURY_ADDRESS`.

## Run order after I build

```text
1. Paste CIRCLE_API_KEY + PRIVY_APP_ID in Project Settings → Secrets
2. node scripts/bootstrap-circle.mjs
3. Paste the 3 printed values back into Secrets
4. Fund printed address at https://faucet.circle.com/ (USDC + EURC + cirBTC)
5. node scripts/deploy-arc.mjs DanceMoveTokens
6. bunx sol-merger contracts/DanceMoveTokens.sol build/flat.sol
   node scripts/verify-arc.mjs build/flat.sol
7. Reload preview — sign in with Google, pick token, log a CID
```
